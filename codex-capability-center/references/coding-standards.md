# Coding Standards

## 通用规则

- 先读现有代码结构，再做最小必要改动。
- 优先复用项目已有框架、工具、服务层和组件。
- 不引入新依赖，除非任务确实需要并经过确认。
- 不提交密钥、生产连接、客户隐私或供应商敏感数据。
- 测试覆盖与风险匹配，重点保护订单、报价、库存、财务、合同和客户信息。

## 后端

- API 字段、schema、服务层、数据库模型保持一致。
- 错误码和错误信息要可被前端和业务理解。
- 涉及订单、库存、报价、支付、退款时关注幂等和一致性。

## 前端

- 内部工具优先清晰、高密度、可扫描、可操作。
- 状态齐全：空、加载、错误、成功、无权限、待确认。
- 不用真实隐私数据做样例。

## 数据库

- 迁移前确认兼容性和回滚思路。
- 不无确认删除或覆盖数据。
- 种子数据使用脱敏样例。
