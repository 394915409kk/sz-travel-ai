# Code Quality Checklist

## 提交前检查

- [ ] 改动范围只覆盖本次任务。
- [ ] 没有提交真实密码、token、cookie、生产连接串或客户隐私。
- [ ] 命名、结构、错误处理符合现有项目风格。
- [ ] 复杂逻辑有必要注释，普通代码不堆注释。
- [ ] 没有绕过认证、授权、审计、库存锁定、财务校验。
- [ ] API、前端、数据库和测试之间字段一致。
- [ ] 已运行相关测试，或说明无法运行的原因。

## Review 重点

- [ ] 是否存在行为回归。
- [ ] 是否有未处理的空值、边界值、异常状态。
- [ ] 是否有并发、重复提交、幂等问题。
- [ ] 是否把业务待确认信息写成确定事实。
- [ ] 是否有无关重构或格式化噪音。
