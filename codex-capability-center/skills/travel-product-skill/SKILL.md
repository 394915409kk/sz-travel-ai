# Travel Product Skill

## 触发场景

- 设计或优化旅游产品、线路、SKU、行程、卖点、适配人群。
- 拆解竞品产品页、OTA 页面、销售资料或私域产品卡。
- 生成产品详情、报价前置资料、行程说明和风险提示。

## 执行流程

1. 确认目的地、出行日期、人数、客群、预算、住宿、交通和强约束。
2. 拆分产品结构：核心卖点、行程节奏、资源组合、费用包含/不含。
3. 标记不确定信息：航班、酒店、签证、库存、价格、政策、保险。
4. 输出销售可用版本，同时给运营和风控保留执行提示。
5. 对需要核价或供应商确认的项目列出清单。

## 禁止事项

- 不编造航班时刻、酒店价格、房态、签证政策、景点开放、保险条款。
- 不承诺“必过签”“绝对无购物”“零风险退款”等无法保证内容。
- 不隐瞒老人、儿童、孕妇、行动不便客户的安全限制。
- 不把营销卖点替代真实履约能力。

## 完成标准

- 产品定位、客群、卖点、行程、费用边界和风险提示完整。
- 所有待核价、待查政策、待确认库存已标记。
- 销售可直接转发，运营知道下一步要补什么。

## 输出格式

```markdown
## 产品方案
- 产品名：
- 目标客群：
- 核心卖点：
- 行程框架：
- 费用包含/不含：
- 风险与待确认：
```
