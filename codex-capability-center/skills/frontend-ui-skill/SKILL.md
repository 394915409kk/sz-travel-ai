# Frontend UI Skill

## 触发场景

- 新建或优化内部页面、运营看板、表单、列表、详情页、弹窗和工作台。
- 修复页面错位、按钮不可用、移动端适配、状态展示或交互问题。
- 将 Figma、截图、文字需求转成可用页面。

## 执行流程

1. 检查现有前端框架、组件库、样式约定和路由结构。
2. 明确目标用户、核心任务、输入输出、空状态、加载态、错误态。
3. 优先复用现有组件和设计模式，保持信息密度适合内部运营。
4. 实现后进行浏览器检查，覆盖桌面和移动关键视口。
5. 检查文字不溢出、控件可点击、状态不重叠、关键流程可完成。

## 禁止事项

- 不做无关营销落地页或装饰性页面。
- 不使用真实客户隐私、真实订单、真实财务数据做前端样例。
- 不以视觉效果牺牲运营效率、可读性和可验收性。
- 不引入新 UI 库，除非项目已有或用户确认。

## 完成标准

- 页面能完成目标工作流。
- 关键状态齐全：空、加载、错误、成功、无权限或待确认。
- 桌面和移动布局没有明显重叠、截断或遮挡。
- 验证方式和未覆盖风险已说明。

## 输出格式

```markdown
## 前端改动
- 页面/组件：
- 用户流程：
- 视觉与交互验证：
- 风险：
- 待确认：
```
