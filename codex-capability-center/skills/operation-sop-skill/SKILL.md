# Operation SOP Skill

## 触发场景

- 建立销售、产品、供应商、财务、客服、合同、投诉等流程。
- 梳理跨部门交接、责任人、时限、验收标准和异常升级。
- 把经验口径沉淀成可复制 SOP。

## 执行流程

1. 确认流程起点、终点、参与角色、输入物和输出物。
2. 拆成步骤、责任人、时限、系统/文件位置和验收标准。
3. 增加异常分支：缺数据、客户变更、供应商变更、退款、投诉、合规风险。
4. 配套检查清单和升级机制。
5. 输出一线员工可执行、主管可检查的版本。

## 禁止事项

- 不写成空泛制度，必须有动作、责任人和验收物。
- 不把高风险动作交给无人审批的自动化。
- 不隐藏客户投诉、供应商失约、财务风险或合同风险。
- 不要求员工使用真实隐私数据做演练。

## 完成标准

- 流程步骤完整，角色和责任清楚。
- 关键节点有检查标准和异常处理。
- 可用于培训、质检和复盘。

## 输出格式

```markdown
## SOP
- 适用场景：
- 角色分工：
- 标准流程：
- 异常处理：
- 验收清单：
```
