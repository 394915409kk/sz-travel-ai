# 05 GitHub / PR 审计

## 定位

本文件用于统一分支、commit、push、PR、Files changed、检查和合并前审计流程。

## 分支前检查

- 工作区是否干净
- 当前仓库和远端是否正确
- 本地 main 是否基于最新 origin/main
- 任务是否要求独立分支
- commit、push、PR、merge 权限是否分别明确

## PR 快速复查

- 标题和范围是否一致
- Files changed 是否只有任务相关文件
- 是否误改业务代码、前端、数据库或部署配置
- 是否包含 `.env`、密钥、token、隐私或业务敏感资料
- 是否包含破坏性 SQL、删除文件或不可逆操作
- 测试和验证结果是否可复现
- 文档是否与代码一致

## 合并前红线

- 不自动 merge。
- 不在检查未完成时建议合并。
- 不以 PR 描述代替实际 diff 检查。
- 不忽略未提交文件、未推送 commit 或分支落后状态。
- 数据库和生产变更必须保留人工确认。

## 审计结论

审计报告只能使用以下结论之一：

- `可以进入人工合并`
- `暂不建议合并，需要修复`
- `信息不足，等待人工确认`
