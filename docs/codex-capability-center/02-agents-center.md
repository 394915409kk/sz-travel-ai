# 02 Agents 中心

## 定位

Agents 中心用于管理角色型数字员工，明确每个 Agent 的职责、输入、输出、权限和交接边界。

## Agent 登记字段

- Agent 名称
- 角色定位
- 负责范围
- 不负责范围
- 可调用 Skills
- 可使用数据
- 输出对象
- 人工审批点
- 异常升级路径
- 当前版本

## 协作原则

- 一个任务只设置一个最终责任 Agent。
- 多 Agent 协作必须明确上游输入和下游输出。
- 分析型 Agent 不自动获得执行权限。
- 涉及财务、合同、客户隐私和生产环境时，必须最小化数据暴露。
- Agent 不得自行扩大任务范围。

## 状态定义

- `草案`：角色说明尚未验收。
- `试运行`：仅用于受控测试。
- `可用`：已通过当前场景验收。
- `限制使用`：存在明确适用边界。
- `停用`：不再用于新任务。
