# 03 SOP 库

## 定位

SOP 库用于沉淀可重复、可检查、可交接的标准作业流程。

## SOP 标准结构

每份 SOP 应包含：

1. 目标与适用范围
2. 前置条件
3. 输入材料
4. 执行步骤
5. 验证方法
6. 风险与红线
7. 失败处理
8. 人工确认节点
9. 输出与归档位置
10. 版本记录

## 建议分类

- 开发与测试
- GitHub 与 PR
- 数据库迁移与恢复
- Skills 安装与升级
- Agent 配置与验收
- 上下文交接
- 故障响应
- 文档维护

## 管理规则

- SOP 必须与实际工具和代码行为一致。
- 未验证步骤必须标记为 `待验证`。
- 高风险 SOP 必须包含停止条件和回滚方式。
- SOP 更新后应同步版本记录。
