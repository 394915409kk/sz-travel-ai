# 08 安全边界

## 总原则

能力中心默认采用最小权限和人工确认机制。没有明确授权时，只允许检查、分析和输出建议。

## 禁止写入仓库的内容

- 真实密码、token、API Key
- 真实生产数据库连接
- 客户隐私和证件资料
- 合同原件和财务凭证
- 供应商底价和付款审批资料
- 未脱敏的生产日志或备份

## Git 安全边界

- 不默认 commit、push、创建 PR、merge 或发布。
- 不直接在 main 上开发。
- 不执行 force push、破坏性 reset 或删除远端分支。
- 每个 Git 动作必须符合用户当次授权范围。

## 数据库安全边界

- 不连接或写入真实生产数据库，除非有明确授权和人工确认。
- 不自动执行 DROP、TRUNCATE、批量 DELETE 或清库操作。
- 迁移前必须检查目标、版本和备份。
- 生产迁移与恢复必须保留人工执行步骤。

## 业务安全边界

- 不编造价格、库存、航班、政策、利润或经营结论。
- 不自动发布内容、联系客户、付款或对外承诺。
- 分析结果必须区分事实、推断和待确认信息。

## 异常处理

发现越权、敏感信息或不可逆风险时，应立即停止，保留证据并请求人工确认。
