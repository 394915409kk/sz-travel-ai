# 00 总控台

## 目标

总控台是能力中心的统一入口，用于接收任务、判断风险、选择能力、分配执行路径并记录最终状态。

## 任务登记

每项任务至少记录：

- 任务名称
- 业务目标
- 目标仓库与工作目录
- 当前分支与基线 commit
- 允许操作
- 禁止操作
- 验收标准
- 人工确认节点

## 路由规则

| 任务类型 | 路由位置 |
|---|---|
| 单一能力调用 | Skills 中心 |
| 多角色协作 | Agents 中心 |
| 固定流程执行 | SOP 库 |
| 安装或升级 | 安装归档 |
| 分支和 PR 检查 | GitHub / PR 审计 |
| 长任务续接 | 上下文交接 |
| 异常和失败 | 故障复盘 |

## 状态定义

- `待确认`：目标或权限尚未确认。
- `可执行`：范围、权限和验收标准已明确。
- `执行中`：正在实施或验证。
- `待人工检查`：自动验证完成，等待人工复核。
- `已完成`：验收通过且记录完整。
- `已阻断`：存在明确阻断条件，暂不能继续。

## 总控红线

总控台不得绕过安全边界，不得把“建议执行”视为“已获授权”，不得替代人工完成 merge、发布和生产数据库写入。
