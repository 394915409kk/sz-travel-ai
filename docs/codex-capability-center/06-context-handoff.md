# 06 上下文压缩与交接

## 定位

本文件用于长任务、跨线程和跨人员协作时保存最小但充分的执行上下文。

## 必须保留的信息

- 用户最终目标
- 当前仓库和工作目录
- 当前分支与最新 commit
- 已完成事项
- 未完成事项
- 已运行验证及结果
- 已发现风险
- 当前限制和禁止操作
- 下一步建议

## 交接模板

```text
任务：
仓库：
分支：
最新 commit：
已完成：
待完成：
验证结果：
已知风险：
禁止操作：
下一步：
```

## 压缩原则

- 保留决策和证据，不保留无用过程噪声。
- 区分已验证事实、推断和待确认事项。
- 不把过期状态写成当前事实。
- 不记录真实密码、token、生产连接或客户隐私。
- 交接后必须重新核对分支、状态和外部系统信息。
