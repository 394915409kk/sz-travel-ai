# Security Boundary

本文件说明当前系统的安全边界。当前版本目标是 Internal Beta Ready，不是正式生产系统。

## 当前已具备

1. 内测级 API Key 鉴权，通过 `X-Internal-API-Key` 保护关键写接口。
2. API Key 只允许通过环境变量 `INTERNAL_API_KEY` 配置。
3. development 且未配置 API Key 时可本地免鉴权。
4. staging/production 环境缺少 API Key 会被健康检查标记为不就绪。
5. 客户姓名、手机号、证件号提供可选脱敏能力。
6. 订单创建、Mock 支付、报价转订单、财务生成和财务状态更新写入操作审计日志。
7. SQLite 提供本地备份和恢复脚本。

## 当前不包含

1. 当前只是内测级 API Key，不是正式用户权限系统。
2. 不包含真实支付、银行、税务、发票。
3. 不包含真实短信、微信、邮件。
4. 不包含真实 OTA、航司、酒店库存。
5. 不包含真实外部 AI API。
6. 不包含客户授权管理、正式登录、RBAC 或租户隔离。
7. 不包含备份加密、日志集中化脱敏或合规审计。

## 内测要求

1. 不允许提交真实密钥。
2. `.env.example` 只能出现示例值。
3. 客户信息需要最小化展示，外发截图或列表应使用 `mask_sensitive=true`。
4. 内测环境建议部署在受控网络或 VPN 后面。
5. 代理层建议保护全部接口，包括只读接口和 Swagger。
6. 备份文件必须保存在受控目录，不得进入公开仓库。

## 正式生产前必须补充

1. 用户登录。
2. RBAC 权限。
3. 更完整的操作审计和审计保留策略。
4. HTTPS 和反向代理安全配置。
5. 数据库加密或云数据库安全策略。
6. 备份加密和备份恢复演练。
7. 日志脱敏。
8. 隐私合规评估。
9. API 频率限制和异常告警。
10. 正式密钥轮换机制。
